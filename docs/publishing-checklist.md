# Publishing Checklist

Use this checklist before making the repository public.

- [ ] Confirm the project is owned by Ahmed or is otherwise allowed to be published.
- [ ] Confirm no employer, client, or private institutional code is included.
- [ ] Confirm images and media assets are allowed to be published.
- [ ] Keep original database dumps out of the repository.
- [ ] Use sanitized demo data if public examples are needed.
- [ ] Rotate any credentials that may have existed in old private versions.
- [ ] Confirm `.env` files are not committed.
- [ ] Run the frontend build with the legacy Node/npm versions documented in the README.
- [ ] Review the README for accurate setup and portfolio positioning.
